# Barber Booking Website

نسخة جاهزة للتجربة محلياً بنفس الواجهة تقريباً، لكن تم تحويل التخزين من Supabase إلى وضعين:

1. وضع تجريبي مباشر بدون أي إعدادات ويستخدم localStorage.
2. وضع إنتاجي باستخدام Firebase Firestore.

## التشغيل المحلي

```bash
npm install
npm run dev
```

ثم افتح الرابط الذي يظهر لك غالباً:

```text
http://localhost:5173
```

## كيف يعمل الآن

- إذا كانت متغيرات Firebase فارغة، يعمل الموقع فوراً في وضع تجريبي محلي.
- إذا أضفت إعدادات Firebase في `.env`، يبدأ استخدام Firestore تلقائياً.
- إذا أضفت إعدادات EmailJS، يتم إرسال إشعار بريد بعد الحجز.

## تفعيل Firebase

1. أنشئ مشروع Firebase.
2. فعّل Firestore Database.
3. انسخ إعدادات الويب إلى `.env`.
4. أنشئ collections باسم:
   - `barbers`
   - `services`
   - `bookings`
5. أو فقط شغّل الموقع مرة واحدة وسيضيف بيانات أولية تلقائياً إذا كانت collections فارغة.
6. طبّق القواعد الموجودة في `firestore.rules`.

## تفعيل الإيميل

هذا المشروع يدعم EmailJS بشكل اختياري.
أضف القيم التالية في `.env`:

- `VITE_EMAILJS_SERVICE_ID`
- `VITE_EMAILJS_TEMPLATE_ID`
- `VITE_EMAILJS_PUBLIC_KEY`
- `VITE_NOTIFICATION_EMAIL`

## ملاحظات

- تم الإبقاء على نفس النمط الداكن والذهبي.
- الحجز يمنع تكرار نفس الوقت لنفس الحلاق.
- عند إلغاء الحجز يصبح الوقت متاحاً مرة أخرى.
- تبويب "حجوزاتي" يبحث برقم الهاتف بدل الاسم لخصوصية أفضل.
- تبويب "الحجوزات القادمة" يعرض كل الحجوزات المؤكدة.

## البناء للإنتاج

```bash
npm run build
```

ينتج مجلد `dist` الجاهز للرفع.
